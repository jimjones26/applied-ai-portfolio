I haven't hand-written a line of code in two years. Across that span I've directed AI through ~83 projects — refining fuzzy ideas into testable problem statements, exploring architectures in parallel, producing implementation plans, and executing them through orchestrator/worker patterns and custom Claude Code skills. The work is the orchestration: where the LLM can hold the architecture, where it can't, what review strategy each stack demands, and how to draw the boundary between deterministic spine and non-deterministic leaves so the system stays correct.

The practice spans stacks I review fluently — JavaScript, TypeScript, SvelteKit — and stacks I don't write but have shipped through AI orchestration: Rust, Go, Java + Spring Boot, Unity C#, Python. The eight case studies below each pick one specific lesson from that body of work: what broke, what held, and what it means for putting AI in production.

#02

Two Experiments in "Let the LLM Run It"

Two architectures that pushed work onto the LLM directly. One built an orchestrator and 14 subagent definitions on top of a claude --subagent=<name> bash invocation that turned out not to exist in that form. The other tried plain-English CLAUDE.md rules ("do not write files directly") as runtime constraints; the LLM treated them as suggestions and wrote files anyway. Text instructions aren't enforcement boundaries, and runtime primitives need validation before architecture commits to them.

#03

Hallucination Needs Its Own Pipeline Stage

Rebuilding a business website section-by-section with an LLM surfaced two failure modes prompt engineering couldn't close: the model invented details that weren't on the source site, and sections generated in separate calls didn't structurally compose. Each needed its own dedicated pipeline stage — a fact-check pass and a structural-normalization pass. Pipeline length ends up diagnostic: every extra stage is a failure mode you discovered by running, not by designing.

#04

Personal AI Memory: The O(history) Problem

Six iterations of a personal AI assistant that accumulates memory across sessions instead of resetting each time (jarvis-core → neura-core-v3, 15-crate Rust workspace, 404 commits, plus a deployed Python instance). Two memory architectures, both with real costs: replaying a chronicle of past events is auditable but boot time grows with history; storing memory in learned weights avoids that but you can't audit what it remembers. The harder problem turned out to be safety — the local memory store got wiped multiple times in development before I added hooks to intercept irreversible commands.

#05

Active Research: Mixing Determinism and Non-Determinism

An open research thread on how to combine deterministic and non-deterministic execution so an LLM-powered system stays consistent and correct. Two faces: workflow tooling (pi-harness — slash commands that force you to declare what's deterministic before you can write code) and empirical comparison (first-principles — RAG vs. open RLM vs. Lambda-RLM against the same test suite). The pattern emerging: deterministic spine, LLM only at the leaves, boundary declared explicitly. About to be field-tested in real-transactions Phase 2 — a real-estate transaction coordinator where uncontrolled LLM non-determinism isn't acceptable.

#06

AI Development Methodology at Scale

Two AI-orchestrated development methodologies — process constraints, slash commands, prompt templates — built to test how far solo + LLM can scale. The 9-module TypeScript version (MAOD) finished cleanly. The 21-microservice Go version (MOAD) stalled at 1 service implemented. Same methodology, different module count. The ceiling isn't LLM capability — it's that the interface contracts between modules have to fit in the LLM's working context to stay coherent, and at 21 they don't.

#07

Production AI Agents: Claude Agent SDK Stability as an Architectural Risk

A complete multi-tenant lead-response system for HVAC contractors: a webhook receives a lead, an AI agent qualifies it over SMS (Vonage), books an appointment on Google Calendar, and a live dashboard streams metrics. Backend, frontend, and infrastructure all built — but never run against real credentials in production. Two findings: the Claude Agent SDK was changing fast enough that web-search results were consistently wrong, so I bundled the official docs locally as a first-class dependency. And the stateful agent layer (core.py, 25.5KB) is the largest file in the backend — managing conversation state, retries, and tool-use constraints is more code than the business logic it serves.

#08

Browser Automation Against Adversarial Targets

A procurement tool that fired a swarm of browser-automation agents at electronics supplier portals (DigiKey, LCSC, Mouser, Arrow) in parallel, then ranked the results with an LLM. Built end-to-end for the TinyFish Accelerator demo. Failed in the wild because Cloudflare bot protection broke agents non-deterministically — works-90%-then-silently-fails behavior, with no fix short of supplier API access (which suppliers don't offer). The pivot to Argus — the same browser-automation skill aimed at public business-intelligence sources that want to be automated — was the correct move.
